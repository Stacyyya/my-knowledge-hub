# Knowledge Hub — 개인 지식창고 & 포트폴리오 사이트 설계

작성일: 2026-06-20
작성자: Stacy Kim (DS팀 시니어 디자이너)

## 목적 (Why)

내가 만든 것들을 한 곳에 잘 **모으는 아카이브**이자, 자랑/평판/협업으로 이어질 수 있는 **포트폴리오**. 목적 우선순위는 "반반(C)" — 잘 모으되, 보여줄 만한 건 예쁘게 보여준다.

세 가지 핵심 욕구:
1. 내가 만든 스킬을 올리고, 다른 사람이 가져갈 수 있게 한다 (공유 + 자랑)
2. 데일리 바이브코딩 챌린지 산출물을 쇼케이스한다 (모으기가 핵심)
3. "나는 누구이고 무엇을 만드는 사람인가"를 보여주는 입구를 둔다

## 범위 (Scope)

**v1에 포함:**
- 홈/About 페이지
- Skills 섹션 (목록 + 상세, 파일 다운로드)
- Vibe Coding 섹션 (갤러리 + 상세)

**v1에서 제외 (나중에):**
- 콘텐츠 필드 추가/수정 (난이도, 카테고리 등 — 운영하며 조정)
- 댓글/방문자 상호작용, 검색, 태그 필터, 조회수
- 외부 도구(Notion 등) 연동
- 다국어

## 사용자 / 기술 전제

- 사용자는 코딩 경험이 거의 없음 → 새 항목 추가는 Claude와 대화로 처리. 비개발자 친화적 안내 필요.
- 콘텐츠는 **파일 기반**(마크다운). 파일 추가 = 사이트에 자동 반영. 모든 방문자에게 동일하게 보이는 진짜 아카이브.
- 공개 사이트이므로 사내 민감정보를 콘텐츠/메모에 넣지 않는다.

## 콘텐츠 관리 방식

- 콘텐츠는 리포지토리 안의 마크다운 파일로 존재한다. localStorage 미사용(공개 일관성/백업을 위해).
- 사용자가 올리고 싶은 것만 **직접 큐레이션**한다. `~/.claude/skills`의 모든 스킬을 자동 수집하지 않음 — 파일을 추가한 스킬만 사이트에 등장.

## 사이트 구조

```
홈/About  ──→  Skills (목록 → 상세, .md/.zip 다운로드)
   │
   └────────→  Vibe Coding (갤러리 → 상세: 이미지·링크·회고)
```

페이지:
1. **홈/About** (`/`) — 첫인상. "나는 누구 + 무엇을 모으는가" + 두 섹션 입구. 각 섹션 항목 개수 표시.
2. **Skills 목록** (`/skills`) — 카드/리스트.
3. **Skill 상세** (`/skills/[slug]`) — 설명 본문 + 다운로드 버튼.
4. **Vibe 갤러리** (`/vibe`) — 썸네일 그리드, 최신/며칠차 순.
5. **Vibe 상세** (`/vibe/[slug]`) — 제목·날짜·이미지·설명·링크·회고.

## 콘텐츠 데이터 모델

### Skills (`src/content/skills/<slug>.md`)
```yaml
---
title: humanize-korean
summary: AI 티 나는 한글을 자연스럽게 다듬기
tags: [글쓰기, 한글]
download: /skills/humanize-korean.zip   # 선택: 부속 파일 있을 때
---
(본문) 이 스킬이 뭘 하는지, 언제 쓰면 좋은지...
```
다운로드 자산: `public/skills/<slug>.zip` (또는 `.md`).
공유 방식 = **파일 다운로드(B)**. 복사 버튼은 v1 제외.

### Vibe Coding (`src/content/vibe/<slug>.md`)
```yaml
---
title: 유아식 레시피 앱
day: 12                 # 며칠차 챌린지
date: 2026-06-07
cover: /vibe/001-recipe-app/cover.png
liveUrl: https://...    # 선택
repoUrl: https://...    # 선택
tools: [Claude Code, Figma]
oneLiner: 냉장고 재료로 윤슬 메뉴 추천
---
(본문) 회고 — 이번엔 ~를 배웠다   # 선택 입력
```
이미지 자산: `public/vibe/<slug>/`.

## 디자인 시스템

룩앤필: **다크 터미널 + 모노스페이스 + 극단적 타이포 크기 대비**. 흔한 AI 룩(보라 그라데이션·글래스모피즘·중앙정렬 SaaS)은 의도적으로 배제.

**컬러 토큰**
| 역할 | 값 |
|---|---|
| 배경 | `#0c0c0c` |
| 본문 | `#ededed` |
| 흐린 글씨 | `#6e6e6e` |
| 포인트(주황) | `#f0883e` |
| 경계선 | `#2a2a2a` |

**타이포그래피**
- 폰트: 모노스페이스 웹폰트(JetBrains Mono 등) + 한글 모노 폴백.
- 3단 리듬: 거대 디스플레이(52–80px, 자간 좁게) ↔ 작은 라벨(10–11px, 대문자·자간 넓게) ↔ 작은 본문.

**반복 요소(컴포넌트)**
- 인덱스 번호 라벨 (`001 —`)
- 화살표 + 개수 (`→ SKILLS 12`)
- 썸네일 카드 (Vibe 갤러리)
- 굵은 보더 구분선

**반응형**: 모바일 대응 필수.

## 컴포넌트 분해 (각자 한 가지 역할)

- `Layout.astro` — 공통 셸(색/폰트/메타/푸터). 모든 페이지가 사용.
- `IndexLabel` — `001 —` 형태 라벨. 입력: 번호, 텍스트.
- `SectionLink` — `→ SKILLS 12` 입구 링크. 입력: 라벨, href, 개수.
- `SkillCard` / `SkillList` — 스킬 목록 표현.
- `VibeCard` / `VibeGallery` — 바이브 썸네일 그리드.
- 상세 페이지는 콘텐츠 컬렉션 항목을 받아 렌더.

각 컴포넌트는 props로만 소통하고 독립적으로 이해/테스트 가능해야 한다.

## 성공 기준

- 파일 하나 추가로 새 스킬/바이브 항목이 사이트에 등장한다.
- 방문자가 스킬을 다운로드할 수 있다.
- 홈에서 타이포 리듬이 살아있는 첫인상을 준다 (AI 흔한 룩이 아님).
- 모바일에서 잘 보인다.
- Vercel 공개 URL로 접근된다.

## 열린 항목 (운영하며 결정)

- 복사 버튼, 검색/태그 필터 등 v2 기능.
- 초기에 어떤 스킬·바이브 항목을 시드로 올릴지.
